# Inventory & Order Management System

A simplified, high-fidelity Inventory & Order Management application designed to handle product stock tracking, customer registries, and transactional order fulfillment.

## Tech Stack
- **Backend API**: Python 3.11 with FastAPI (async support, pydantic v2 data validation, and autogenerated OpenAPI documentation).
- **Frontend Dashboard**: React 18 with TypeScript and Vite. Standard responsive CSS styling with dark-mode, glassmorphism design accents.
- **Database**: PostgreSQL 15 (relational structure with transactional operations).
- **Containerization**: Docker & Docker Compose.

---

## Core Features & Business Rules
1. **Unique SKUs**: Product SKUs are unique identifiers. Any attempt to create or modify a product with a conflicting SKU returns a `400 Bad Request`.
2. **Unique Customer Emails**: Emails are validated via `EmailStr` and must be globally unique across all customers.
3. **Transactional Order Validation**: Placing an order locks the involved product rows (`SELECT FOR UPDATE`) to prevent race conditions during concurrent stock updates:
   - Check current available stock levels.
   - Fail transaction with a detailed error message if stock is insufficient.
   - Atomically deduct inventory items and log the completed order.
4. **Order Restocking (Cancellation)**: Cancelling an order updates its status to `Cancelled` and automatically restores the previously deducted stock levels to the respective products.

## Getting Started

### Prerequisites
- [Docker](https://www.docker.com/products/docker-desktop) and Docker Compose installed.

### Option 1: Run with Docker Compose (Recommended)
1. In the root directory, configure the environment variables:
   ```bash
   cp .env.example .env
   ```
2. Start the application services:
   ```bash
   docker compose up --build
   ```
3. Once running, open:
   - **Frontend App**: [http://localhost](http://localhost)
   - **API Docs (Swagger)**: [http://localhost:8000/docs](http://localhost:8000/docs)
   - **API Health Check**: [http://localhost:8000/api/health](http://localhost:8000/api/health)

*Note: On initial startup, if the database has no contents, it is automatically seeded with sample products, customers, and a history transaction to let you preview the dashboard immediately.*

## API Endpoints Reference

### Dashboard
- `GET /api/dashboard/stats` - Fetch overall revenues, inventory flags, catalogs, and recent activity.

### Products
- `GET /api/products?search=<str>` - List and filter catalog products.
- `GET /api/products/{id}` - Fetch detail profile.
- `POST /api/products` - Create product (requires unique SKU).
- `PUT /api/products/{id}` - Update properties.
- `DELETE /api/products/{id}` - Delete product.

### Customers
- `GET /api/customers?search=<str>` - Search customers.
- `GET /api/customers/{id}` - Profile details.
- `POST /api/customers` - Register a customer (requires unique email).
- `PUT /api/customers/{id}` - Modify details.
- `DELETE /api/customers/{id}` - Remove customer.

### Orders
- `GET /api/orders` - View all historical transactions.
- `GET /api/orders/{id}` - View order details (items, prices).
- `POST /api/orders` - Place order (requires stock check and deduction).
- `POST /api/orders/{order_id}/cancel` - Cancel order and restore product stock levels.

## Public Deployment Guide

To deploy this project to public hosting solutions for free, follow these recommended paths:

### 1. Database (PostgreSQL)
Deploy the database on **Supabase** or **Neon**:
- Create a free project.
- Retrieve the connection string: `postgresql://[user]:[password]@[host]:5432/[dbname]?sslmode=require`.

### 2. Backend (FastAPI)
Deploy on **Render** or **Railway**:
- Link your GitHub repository.
- Choose **Web Service** with build command `pip install -r backend/requirements.txt` and start command `uvicorn backend.app.main:app --host 0.0.0.0 --port $PORT` (adjust paths if deployed from root or configure custom build directory).
- Set Environment Variables:
  - `DATABASE_URL` pointing to your Neon/Supabase instance.
  - `ENVIRONMENT` to `production`.

### 3. Frontend (React)
Deploy on **Vercel** or **Netlify**:
- Link your GitHub repository.
- Set the build settings:
  - Framework Preset: `Vite` (or `Other` if auto-detected).
  - Root directory: `frontend`.
  - Build command: `npm run build`.
  - Output directory: `dist`.
- Set Environment Variables:
  - If you deploy your API on Render, you should configure Vite to query that API. You can modify `API_BASE` in `App.tsx` or build a `.env` file in the frontend with `VITE_API_URL` pointing to your public backend URL.
- To handle Nginx-like client-side routing on Vercel, add a `vercel.json` rewrite in the frontend root:
  ```json
  {
    "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
  }
  ```
